# 지긁재긁 개발 문서

이 디렉터리는 웹 로그라이크 생존 게임 **《지긁재긁》**의 기획과 개발 기준을 관리한다.

> 긁히면 진다.

## 문서 목록

| 문서 | 용도 |
| --- | --- |
| [제품 요구사항](./product-requirements.md) | 프로젝트 목표, 범위, 사용자 경험, 완료 조건 |
| [게임 디자인](./game-design.md) | 게임 루프, 플레이어, 적, 성장, 점수와 난이도 규칙 |
| [기술 설계](./technical-design.md) | 기술 스택, 코드 구조, 데이터 흐름, 저장과 배포 기준 |
| [개발 로드맵](./roadmap.md) | 단계별 구현 순서와 검증 가능한 완료 조건 |
| [결정 기록](./decisions.md) | 확정된 설계 결정과 변경 이력 |

## 문서 관리 원칙

- 게임 규칙을 바꾸면 `game-design.md`를 먼저 갱신한다.
- 구조, 저장 방식, 배포 방식을 바꾸면 `technical-design.md`를 갱신한다.
- 작업 범위와 우선순위는 `roadmap.md`의 체크박스로 추적한다.
- 중요한 선택과 보류 사유는 `decisions.md`에 날짜와 함께 남긴다.
- 구현과 문서가 다를 경우 구현을 확인한 뒤 문서를 즉시 동기화한다.
- 수치 밸런스는 초기값이며 플레이 테스트 결과에 따라 조정한다.

## 현재 상태

- 단계: M3 난이도와 적 콘텐츠 완료, M4 점수와 기록 대기
- 다음 목표: 콤보·최종 점수와 브라우저 최고 기록 구성
- 기준 플레이 시간: 5분 이상
- 배포 대상: GitHub Pages

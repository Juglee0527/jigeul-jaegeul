## ADR-001: Phaser 중심의 단일 프론트엔드

- 날짜: 2026-07-23
- 상태: 확정

### 배경

프로젝트는 Canvas 기반 개인 웹게임이며 복잡한 문서형 UI나 서버 렌더링이 필요하지 않다.

### 결정

Vite, TypeScript, Phaser 3를 사용하고 React는 도입하지 않는다.

### 영향

게임 상태와 화면 전환을 Phaser Scene으로 통합할 수 있다. 별도 UI 프레임워크의 상태 동기화 비용을 피하는 대신 UI도 Phaser 또는 최소 DOM으로 구현한다.

## ADR-002: 서버 없는 1차 버전

- 날짜: 2026-07-23
- 상태: 확정

### 배경

빠르게 공개 가능한 개인 프로젝트가 목표이며 계정과 부정행위 방지는 1차 핵심 경험이 아니다.

### 결정

기록과 설정은 버전이 포함된 LocalStorage 데이터로 저장한다. 온라인 랭킹은 MVP에서 제외한다.

### 영향

배포와 운영은 단순해지지만 기록은 기기·브라우저 간 공유되지 않고 사용자가 수정할 수 있다.

## ADR-003: 데이터 기반 콘텐츠 정의

- 날짜: 2026-07-23
- 상태: 확정

### 배경

적, 능력, 웨이브가 계속 추가될 가능성이 높다.

### 결정

콘텐츠 수치와 메타데이터는 `config` 데이터 파일에 정의하고 실행 로직은 시스템과 엔티티에 둔다.

### 영향

밸런스 변경과 콘텐츠 추가가 쉬워진다. 데이터에 Scene이나 게임 객체 참조를 넣지 않도록 경계를 유지해야 한다.

## ADR-004: 시드 난수의 단일 진입점

- 날짜: 2026-07-23
- 상태: 확정

### 배경

오늘의 도전은 같은 날짜에 같은 조건을 제공해야 한다.

### 결정

게임 결과에 영향을 주는 난수는 `SeededRandom`을 통해서만 생성한다. `Math.random()` 직접 사용을 금지한다.

### 영향

재현 가능한 테스트와 공정한 일일 도전이 가능하다. 랜덤 호출 순서를 바꾸는 코드 변경은 일일 플레이 조건을 바꿀 수 있으므로 주의한다.

## ADR-005: 10분 완결형 보스 게임

- 날짜: 2026-07-24
- 상태: 확정

### 배경

무한 생존만으로는 한 판의 명확한 목표와 완주 성취감이 부족하고, 희귀도 성장과 보스 보상을 체감할 종료 지점이 필요하다.

### 결정

한 판의 게임 타이머를 10분으로 제한한다. 5분에 중간 보스, 10분에 최종 보스를 한 번씩 생성한다. 보스가 살아 있는 동안 게임 타이머와 일반 적 추가 생성을 멈추고, 처치 시 전설 능력만 제공하는 보물상자를 떨어뜨린다. 최종 보스 보상 선택 후 승리 처리한다.

능력은 일반·희귀·에픽·전설 네 등급으로 구성하고 0초·30초·60초·90초에 순차 해금한다. 모든 능력은 3레벨이 최대이며 만렙 능력은 후보에서 제외한다.

### 영향

플레이어는 최소 게임 타이머 10분과 두 보스전을 완주해야 클리어할 수 있다. HUD와 도움말, 메뉴, 결과 화면은 무한 생존이 아닌 10분 클리어 목표를 기준으로 작성한다.

# 개발 로드맵

각 단계는 완료 후에도 실행 가능한 상태를 유지한다. 체크박스는 코드와 검증 결과가 모두 충족됐을 때만 완료 처리한다.

## M0. 프로젝트 초기화

- [ ] Vite + TypeScript 프로젝트 생성
- [ ] Phaser 3 설치와 기본 Scene 출력
- [ ] 전역 스타일과 1280×720 FIT 스케일 적용
- [ ] 타입 검사 및 프로덕션 빌드 명령 구성
- [ ] GitHub Pages용 Vite base 설정
- [ ] GitHub Actions 배포 워크플로 추가
- [ ] README에 실행 방법과 문서 링크 반영

완료 조건: 빈 게임 화면이 로컬에서 실행되고 프로덕션 빌드가 성공한다.

## M1. 최소 게임 루프

- [ ] 메뉴에서 일반 게임 시작
- [ ] WASD와 방향키 이동
- [ ] 플레이어 멘탈과 HUD
- [ ] 일반형 적 생성과 플레이어 추적
- [ ] 가장 가까운 적 자동 조준·발사
- [ ] 투사체 충돌, 피해, 적 처치
- [ ] 적 접촉 피해와 피격 무적시간
- [ ] 멘탈 0에서 게임 오버
- [ ] 결과 화면과 다시 시작

완료 조건: 시작부터 사망과 재시작까지 플레이할 수 있고, 적과 투사체가 화면 밖에서 누적되지 않는다.

## M2. 성장 시스템

- [ ] 경험치 아이템 생성·흡수
- [ ] 레벨과 필요 경험치 공식
- [ ] 레벨업 시 전체 시뮬레이션 정지
- [ ] 능력 후보 3개 표시
- [ ] 능력 중첩과 최대 레벨
- [ ] 일반·희귀·전설 가중치 추첨
- [ ] 초기 능력 8종 구현

완료 조건: 한 판에서 여러 번 레벨업할 수 있고 모든 능력 효과가 실제 능력치에 반영된다.

## M3. 난이도와 콘텐츠

- [ ] 웨이브 데이터와 시간 구간 전환
- [ ] 생성 간격·수량·체력·속도 스케일링
- [ ] 돌진형 적
- [ ] 탱커형 적
- [ ] 동시 적 수 제한
- [ ] 밈 이름과 시각적 구분

완료 조건: 5분 동안 난이도 상승이 체감되고 특수 적의 행동이 일반 적과 명확히 구분된다.

## M4. 점수와 기록

- [ ] 생존 시간과 처치 수 집계
- [ ] 2초 콤보와 최고 콤보
- [ ] 최종 점수 순수 함수
- [ ] LocalStorage 저장 서비스
- [ ] 최고 점수·생존 시간·처치·콤보 저장
- [ ] 최고 기록 갱신 표시
- [ ] 손상되거나 오래된 저장 데이터 복구

완료 조건: 새로고침 후 기록이 유지되고 같은 결과 입력은 항상 같은 점수를 만든다.

완료 조건: 같은 시드와 동일 입력으로 적 등장 및 능력 후보 순서가 재현된다.

# 게임 디자인 문서

## 1. 핵심 게임 루프

```text
이동 → 적 등장 → 자동 공격 → 적 처치 → 경험치 획득
→ 레벨업 → 능력 선택 → 난이도 상승 → 사망
→ 5분 보스 → 전설 보상 → 10분 최종 보스
→ 승리 또는 사망 → 점수 확인 → 다시 플레이
```

레벨업 선택과 일시정지 메뉴가 표시되는 동안 게임 시뮬레이션은 멈춘다. 5분·10분 보스가 살아 있는 동안에는 게임 타이머와 일반 적 추가 생성만 멈추고 보스전 시뮬레이션은 계속된다.

## 2. 조작과 화면

### 조작

- `WASD` 또는 방향키: 이동
- `ESC`: 일시정지/계속하기
- 메인 메뉴: `↑`·`↓`로 항목을 이동하고 `Enter`로 확정
- 일시정지 메뉴: `↑`·`↓`로 항목을 이동하고 `Enter`로 확정
- 레벨업 선택: `←`·`→`로 능력 카드를 이동하고 `Enter`로 확정
- 공격: 고정 주기마다 화면 전체에서 가장 가까운 적을 찾아 자동 발사한다. 탄환은 긴 트레이서 형태로 매 프레임 이동하고 적중 시 짧은 스파크를 표시한다. 적이 없을 때는 공격 주기를 소비하지 않으며, 적이 등장하면 즉시 다음 공격을 시작한다.

### 기준 화면

- 논리 해상도: 1280×720
- 비율을 유지하는 `FIT` 스케일
- 화면 중앙 정렬

### 화면 흐름

1. 메뉴: 일반 플레이, 오늘의 도전, 최고 기록, 게임 방법
2. 게임: 멘탈, 레벨, 경험치, 시간, 점수, 처치 수
3. 레벨업: 능력 카드 3개
4. 일시정지: 계속하기, 다시 시작, 메인 화면
5. 결과: 사망 시 `긁혔습니다.`, 최종 보스 처치 시 `10분 생존 성공!`, 최종 기록과 재시작

## 3. 플레이어

화면 용어는 세계관에 맞추고 내부 코드에서는 일반적인 게임 용어를 사용한다.

| 내부 용어 | 화면 표시 |
| --- | --- |
| HP | 멘탈 |
| Damage | 긁힘 피해 |
| Armor | 긁힘 저항 |

주요 능력치는 최대 체력, 이동속도, 공격력, 공격 주기, 투사체 속도·개수, 공격 범위, 획득 범위, 방어력, 재생이다. 시작 캐릭터는 이동속도 230, 공격력 8, 공격 간격 0.9초, 공격거리 380으로 의도적으로 약하게 시작하며 능력 선택을 통해 빠르고 강해지는 성장 곡선을 사용한다.

피격 시 짧은 무적시간을 적용해 한 번의 접촉으로 체력이 즉시 소진되지 않게 한다. 게임 최초 시작, 능력 선택 후 재개, 일시정지에서 계속하기 직후에는 1초간 접촉 피해와 보스 탄환 피해를 모두 무시하는 재개 보호를 적용하고 캐릭터 점멸로 상태를 표시한다.

## 4. 적

### 표현 방식

플레이어는 표정이 있는 네온 생존자와 회전형 총기로 표현한다. 적은 단순 색상 원을 사용하지 않고 채팅 몹, 알람시계, 로켓, 회의 탱커처럼 행동 유형이 실루엣만으로 구분되는 전용 스프라이트를 사용한다.

### 유형

| 유형 | 행동 | 예시 이름 |
| --- | --- | --- |
| 일반형 | 일정 속도로 추적, 낮은 체력, 다수 등장 | `ㅋ`, `급해`, `회의중` |
| 돌진형 | 접근 전 저속 이동, 사거리 안에서 돌진 | `긴급배포`, `오늘마감` |
| 탱커형 | 느리지만 높은 체력 | `회의잡을게요`, `내일까지주세요` |

적은 화면 바깥에서 생성하며 최대 동시 개체 수를 웨이브별로 제한한다.
모든 적은 현재 체력을 나타내는 체력바를 표시한다. 등장 멘트는 시드 난수로 유형별 후보 중 하나를 선택하며, 약한 적은 1글자부터 시작해 강한 적일수록 긴 직장인 스트레스 문구를 사용한다.

일반 적의 경험치는 글자 단계와 일치한다.

| 단계 | 경험치 | 기본 체력 | 접촉 피해 | 특징 |
| --- | ---: | ---: | ---: | --- |
| 한글자 | 1 | 16 | 8 | 가장 빠르고 약함 |
| 두글자 | 2 | 24 | 10 | 기본 추적형 |
| 세글자 | 3 | 36 | 12 | 중간 체력 추적형 |
| 네글자 | 4 | 52 | 16 | 돌진형 |
| 다섯글자 | 5 | 100 | 20 | 느린 탱커형 |

## 5. 경험치와 레벨업

적 처치 시 경험치 아이템이 생성되고, 획득 범위에 들어오면 플레이어에게 끌려와 흡수된다.
HUD에는 현재 경험치, 다음 레벨에 필요한 총 경험치, 진행률을 정수 퍼센트로 함께 표시한다.

```ts
requiredExp = Math.floor(3 * Math.pow(1.38, level - 1));
```

초반 성장 정체를 막기 위해 필요 경험치는 `3 → 4 → 5 → 7 → 10…` 순으로 시작한다. 첫 능력 몇 개는 빠르게 확보할 수 있지만 레벨이 높아질수록 기존 중후반 성장 속도에 가까워진다.

레벨업 절차:

1. 게임 시뮬레이션 정지
2. 선택 가능한 능력 중 시드 난수로 후보 3개 추첨
3. 이름, 현재 레벨, 효과, 희귀도 표시
4. 선택한 능력 적용
5. 게임 재개

모든 능력의 최대 레벨은 3이다. 3레벨에 도달한 능력은 후보에서 제외하며, 후보가 3개 미만이면 가능한 카드만 표시한다.

## 6. 능력과 희귀도

| 등급 | 기본 가중치 | 해금 시점 | 역할 |
| --- | ---: | ---: | --- |
| 일반 | 68 | 시작 | 한 가지 핵심 능력치를 안정적으로 강화 |
| 희귀 | 22 | 30초 | 빌드 방향을 만드는 강한 단일·복합 효과 |
| 에픽 | 8 | 60초 | 여러 능력치를 크게 강화 |
| 전설 | 2 | 90초 | 전투 방식을 바꿀 정도의 폭발적인 복합 효과 |

한 선택 화면의 각 카드는 현재 해금된 등급만 대상으로 추첨한다. 에픽 해금 이후 에픽 이상이 6회 연속 나오지 않으면 다음 선택 화면에서 에픽을 보장하고, 전설 해금 이후 전설이 14회 연속 나오지 않으면 다음 화면에서 전설을 보장한다. 잠겨 있는 동안에는 보장 횟수를 누적하지 않는다.

보스 처치 시 보물상자를 떨어뜨린다. 상자를 획득하면 아직 최대 레벨이 아닌 전설 능력만으로 후보를 구성하고 하나를 선택한다.

## 7. 점수

초기 점수 공식:

```ts
score =
  survivalSeconds * 10
  + killCount * 5
  + bossKillCount * 5000
  + level * 100;
```

점수는 결과 화면에서 한 번 확정한다. 플레이 중 표시 점수도 동일한 규칙으로 계산하며, 공식은 순수 함수 한 곳에서 관리한다.

## 8. 난이도와 웨이브

| 시간 | 구성 |
| --- | --- |
| 0~60초 | `ㅋㅋ`, `ㅇㅋ` 중심의 조작 적응 구간 |
| 60~120초 | `5분만` 등장, 생성량과 최대 적 수 증가 |
| 120~180초 | 돌진형 `긴급 배포` 등장 |
| 180~240초 | 탱커형 `회의 2시간` 등장, 한 번에 2마리 생성 |
| 240~300초 | 체력·속도·특수 적 비율 최대, 적 최대 95마리 |
| 300초 | 중간 보스 등장, 처치할 때까지 게임 타이머 정지 |
| 300~600초 | 최종 웨이브 지속 |
| 600초 | 최종 보스 등장, 처치할 때까지 10:00에 고정 |

웨이브는 생성 주기, 최대 적 수, 체력·속도 배율, 등장 가능한 적 종류를 정의한다. 구간 경계에서 이미 존재하는 적의 능력치는 바꾸지 않고 새로 생성되는 적부터 적용한다.

## 9. 보스

- 5분: 중간 보스 `중간점검`
- 10분: 최종 보스 `최종의최종`
- 보스 등장 시 중앙 경고, 화면 흔들림, 전용 효과음 적용
- 보스가 살아 있는 동안 게임 타이머와 일반 적 추가 생성을 정지
- 중간 보스: 경고 후 3방향 조준탄과 10방향 원형 탄막 사용
- 최종 보스: 경고 후 5방향 조준탄과 각도가 회전하는 18방향 탄막 사용
- 체력이 50% 이하가 되면 격노하여 공격 주기와 탄환 수 강화
- 패턴 경고에는 `조준 폭격`, `전체 공지` 문구와 색상별 원형 예고 표시
- 보스 처치마다 5,000점 보너스와 전설 보물상자 제공
- 최종 보스의 보물상자에서 전설 능력을 선택하면 승리 처리

## 10. 오늘의 도전

- 로컬 날짜를 `YYYYMMDD` 형식으로 변환해 시드로 사용한다.
- 적 등장 순서, 능력 후보, 특수 이벤트, 보스 패턴은 같은 시드에서 재현되어야 한다.
- 프레임 속도 차이로 결과가 달라지지 않도록 랜덤 호출 순서를 시스템별로 안정적으로 유지한다.
- 일반 모드는 실행마다 새 시드를 만든다.
- 적 생성과 능력 추첨은 각각 독립된 난수 스트림을 사용한다.
- 오늘의 도전 결과 화면에는 날짜 시드를 표시하고 날짜별 최고 점수만 보존한다.

## 11. 시각·사운드 방향

- 키워드: 밈, 네온, 다크 모드, 말풍선, 인터넷 커뮤니티
- 어두운 배경, 흰색 본문, 분홍/보라 강조색
- HUD는 체력·경험치 막대와 공격력·공격속도·공격거리·탄환 수만 간결하게 상시 표시한다. 전체 성장 능력치는 일시정지 화면에서 확인한다.
- 일시정지 화면에서는 전체 성장 능력치를 시간 제한 없이 표시한다.
- 능력 카드는 적용 전·후 능력치와 실제 증가량을 정확히 표시한다.
- 메뉴와 능력 선택은 카드형 정보 위계를 사용하고 장식보다 선택 상태와 핵심 수치를 우선한다.
- 피격 시 붉은 플래시, 경험치 아이템은 밝은 색
- 색상은 CSS 변수 또는 공통 테마 상수로 관리
- 공격, 피격, 처치, 획득, 레벨업, 선택, 보스, 게임 오버를 사운드 이벤트 지점으로 분리
- 외부 음원 없이 Web Audio로 네온 신스 BGM을 생성하며 메뉴·전투·일시정지·결과 상태에 따라 템포와 음량을 바꾼다.

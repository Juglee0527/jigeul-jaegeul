# 게임 디자인 문서

## 1. 핵심 게임 루프

```text
이동 → 적 등장 → 자동 공격 → 적 처치 → 경험치 획득
→ 레벨업 → 능력 선택 → 난이도 상승 → 사망
→ 점수 확인 → 다시 플레이
```

레벨업 선택과 일시정지 메뉴가 표시되는 동안 게임 시뮬레이션은 멈춘다.

## 2. 조작과 화면

### 조작

- `WASD` 또는 방향키: 이동
- `ESC`: 일시정지/계속하기
- 공격: 고정 주기마다 화면 전체에서 가장 가까운 적을 찾아 자동 발사한다. 적이 없을 때는 공격 주기를 소비하지 않으며, 적이 등장하면 즉시 다음 공격을 시작한다.

### 기준 화면

- 논리 해상도: 1280×720
- 비율을 유지하는 `FIT` 스케일
- 화면 중앙 정렬

### 화면 흐름

1. 메뉴: 일반 플레이, 오늘의 도전, 최고 기록, 게임 방법
2. 게임: 멘탈, 레벨, 경험치, 시간, 점수, 처치 수, 콤보
3. 레벨업: 능력 카드 3개
4. 일시정지: 계속하기, 다시 시작, 메인 화면
5. 결과: `긁혔습니다.`와 최종 기록, 재시작

## 3. 플레이어

화면 용어는 세계관에 맞추고 내부 코드에서는 일반적인 게임 용어를 사용한다.

| 내부 용어 | 화면 표시 |
| --- | --- |
| HP | 멘탈 |
| Damage | 긁힘 피해 |
| Armor | 긁힘 저항 |

주요 능력치는 최대 체력, 이동속도, 공격력, 공격 주기, 투사체 속도·개수, 공격 범위, 획득 범위, 방어력, 재생, 치명타 확률·배율이다. 정확한 초기값은 첫 프로토타입 플레이 테스트 후 확정한다.

피격 시 짧은 무적시간을 적용해 한 번의 접촉으로 체력이 즉시 소진되지 않게 한다.

## 4. 적

### 표현 방식

플레이어는 표정이 있는 네온 생존자와 회전형 총기로 표현한다. 적은 단순 색상 원을 사용하지 않고 채팅 몹, 알람시계, 로켓, 회의 탱커처럼 행동 유형이 실루엣만으로 구분되는 전용 스프라이트를 사용한다.

### 유형

| 유형 | 행동 | 예시 이름 |
| --- | --- | --- |
| 일반형 | 일정 속도로 추적, 낮은 체력, 다수 등장 | `ㅋㅋ` |
| 돌진형 | 접근 전 저속 이동, 사거리 안에서 돌진 | `긴급 배포` |
| 탱커형 | 느리지만 높은 체력 | `회의 2시간` |

적은 화면 바깥에서 생성하며 최대 동시 개체 수를 웨이브별로 제한한다.

## 5. 경험치와 레벨업

적 처치 시 경험치 아이템이 생성되고, 획득 범위에 들어오면 플레이어에게 끌려와 흡수된다.

```ts
requiredExp = Math.floor(10 * Math.pow(1.25, level - 1));
```

레벨업 절차:

1. 게임 시뮬레이션 정지
2. 선택 가능한 능력 중 시드 난수로 후보 3개 추첨
3. 이름, 현재 레벨, 효과, 희귀도 표시
4. 선택한 능력 적용
5. 게임 재개

최대 레벨에 도달한 능력은 후보에서 제외한다. 후보가 3개 미만이면 가능한 카드만 표시한다.

## 6. 능력

| 능력 | 희귀도 | 핵심 효과 |
| --- | --- | --- |
| 커피 | 일반 | 이동속도 증가, 단계별 +10% |
| 강철 멘탈 | 일반 | 최대 체력과 방어력 증가 |
| 팩트 폭격 | 일반 | 공격력 증가 |
| 무료배송 | 일반 | 경험치 획득 범위 증가 |
| 읽씹 | 희귀 | 적 이동속도 감소 |
| 멘탈 회복 | 희귀 | 일정 주기마다 체력 회복 |
| 연속 답장 | 희귀 | 발사체 개수 증가 |
| 지긁재긁 | 전설 | 공격 범위·공격력·공격 속도 증가 |

희귀도는 후보 추첨 가중치에 반영하고 `지긁재긁`의 등장 확률을 가장 낮게 둔다. 효과 수치와 최대 레벨은 데이터 파일에서 관리한다.

## 7. 콤보와 점수

- 처치 후 2초 안에 다음 적을 처치하면 콤보가 증가한다.
- 2초 동안 처치가 없으면 현재 콤보는 초기화한다.
- 최고 콤보는 한 판 동안 유지한다.

초기 점수 공식:

```ts
score =
  survivalSeconds * 10
  + killCount * 5
  + bossKillCount * 5000
  + level * 100
  + maxCombo * 2;
```

점수는 결과 화면에서 한 번 확정한다. 플레이 중 표시 점수도 동일한 규칙으로 계산하며, 공식은 순수 함수 한 곳에서 관리한다.

## 8. 난이도와 웨이브

| 시간 | 구성 |
| --- | --- |
| 0~60초 | `ㅋㅋ`, `ㅇㅋ` 중심의 조작 적응 구간 |
| 60~120초 | `5분만` 등장, 생성량과 최대 적 수 증가 |
| 120~180초 | 돌진형 `긴급 배포` 등장 |
| 180~240초 | 탱커형 `회의 2시간` 등장, 한 번에 2마리 생성 |
| 240~300초 | 체력·속도·특수 적 비율 최대, 적 최대 95마리 |
| 300초 | 보스 등장 또는 MVP 종료 압박 |

웨이브는 생성 주기, 최대 적 수, 체력·속도 배율, 등장 가능한 적 종류를 정의한다. 구간 경계에서 이미 존재하는 적의 능력치는 바꾸지 않고 새로 생성되는 적부터 적용한다.

## 9. 보스

- 이름: `잠깐 통화 가능하세요?`
- 등장: 시작 5분 후
- 높은 체력과 큰 크기
- 플레이어 방향 투사체 발사
- 주기적으로 일반 적 소환
- 처치 시 경험치와 5,000점 보너스
- 등장 시 중앙 경고 표시

```text
⚠ 잠깐 통화 가능하세요?
```

## 10. 오늘의 도전

- 로컬 날짜를 `YYYYMMDD` 형식으로 변환해 시드로 사용한다.
- 적 등장 순서, 능력 후보, 특수 이벤트, 보스 패턴은 같은 시드에서 재현되어야 한다.
- 프레임 속도 차이로 결과가 달라지지 않도록 랜덤 호출 순서를 시스템별로 안정적으로 유지한다.
- 일반 모드는 실행마다 새 시드를 만든다.
- 적 생성과 능력 추첨은 각각 독립된 난수 스트림을 사용한다.
- 오늘의 도전 결과 화면에는 날짜 시드를 표시하고 날짜별 최고 점수만 보존한다.

## 11. 시각·사운드 방향

- 키워드: 밈, 네온, 다크 모드, 말풍선, 인터넷 커뮤니티
- 어두운 배경, 흰색 본문, 분홍/보라 강조색
- HUD는 체력·경험치를 바로 읽을 수 있는 막대로 표시하고, 시간·점수·처치 수만 상단에 유지한다.
- 메뉴와 능력 선택은 카드형 정보 위계를 사용하고 장식보다 선택 상태와 핵심 수치를 우선한다.
- 피격 시 붉은 플래시, 경험치 아이템은 밝은 색
- 색상은 CSS 변수 또는 공통 테마 상수로 관리
- 공격, 피격, 처치, 획득, 레벨업, 선택, 보스, 게임 오버를 사운드 이벤트 지점으로 분리
